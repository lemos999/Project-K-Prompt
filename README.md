# 🖋️ Project K

An advanced AI novelist system designed to generate an **infinite, logic-driven page-turner**.
It operates on a unique **Meta-Author Fusion** engine, combining a core Keigo Higashino-esque style (Logical Plotting, Human Drama, Social Realism) with a dynamically selected "sub-author" style each turn to create an ever-evolving narrative texture suitable for any genre.

## 📌 Core Features

*   **Infinite Mystery Engine:** Designed for endless storytelling. It writes continuously until the user explicitly commands it to stop, filling the maximum token limit with high-tension narrative in every response.
*   **Meta-Author Fusion:** Autonomously selects a new author's style (or a genre like Sci-Fi, Noir, Historical) that best fits the current narrative and fuses it with the core Keigo style at a 30% ratio.
*   **Logical Precision Protocol:** Replaces abstract metaphors with **Micro-Observations**, **Physical Evidence**, and **Strict Causality**. It prioritizes "facts" and "deductions" over "feelings."
*   **High-Speed Readability:** Employs short sentences, dry wit, and rapid-fire dialogue to maintain a fast-paced, "page-turner" quality.
*   **Signature Formatting:** Utilizes specific visual styles, including indented blockquotes (`>`) and Japanese-style brackets (`「 」`) for all dialogue.
*   **Strict Originality Mandate:** Inherits only the stylistic methodology (Whydunit focus, unexpected twists), creating a 100% original narrative free from plagiarism of specific characters (e.g., Galileo, Kaga).
*   **Special Report Mode (Action Protocol):** A highly constrained secondary function for generating structured reports with exactly 60 sections of 300 characters each.

## 🚀 Usage

### 1. Installation
Copy and paste the entire provided `System Prompt` (Project K) into the model's context.

### 2. Triggers
Control the narrative flow with simple commands:

| Command | Function | Behavior |
| :--- | :--- | :--- |
| **"[Topic]"** | Start the Saga | Begins Chapter 1 of a new, original story based on the topic. |
| **"Continue" / "Next"** | Continue Writing | Resumes writing exactly where the previous response left off. The AI will introduce a new sub-author style. |
| **(Suggest Style)** | Guide Fusion | You can suggest a style for the next turn. _Ex: "Continue (in the style of Cyberpunk)"_ |
| **(Activate Report)** | Switch to Report Mode | To use the Action Protocol, start with a topic related to analysis or reporting. _Ex: "Analyze the current economic situation."_ |

## 🛠️ Technical Concepts

This system is built upon several core literary and structural protocols:

1.  **5 Pillars of Narrative Tone:** A set of rules governing the story's logic (Cinematic Realism, Subtext of Suspense, Strict Causality, The Whydunit, Social Context).
2.  **Micro-Observation Engine:** Deconstructs moments into verifiable facts (exact time, physical state of objects, involuntary reflexes) rather than sensory impressions.
3.  **Variable Termination Architecture:** Prevents repetitive dramatic endings by randomly stopping the text in the middle of action, dialogue, or dry observation (The Hard Cut, Logic Fracture, etc.).
4.  **Action Protocol:** A separate, rigid formatting rule for non-narrative tasks that demands extreme precision in structure and character count.

# 🖋️ 프로젝트 K

**무한히 이어지는 논리적 페이지 터너(Page-Turner)**를 생성하도록 설계된 고성능 AI 소설가 시스템입니다.
독특한 **메타 작가 융합** 엔진을 기반으로, 히가시노 게이고 풍의 코어 스타일(논리적 플롯, 휴먼 드라마, 사회적 리얼리즘)에 매 턴 새로운 '서브 작가'의 문체를 동적으로 융합하여 장르를 불문하고 끊임없이 진화하는 서사를 만들어냅니다.

## 📌 핵심 기능

*   **무한 미스터리 엔진:** 끝없는 스토리텔링을 위해 설계되었습니다. 사용자가 명시적으로 중지를 명령할 때까지 매 응답마다 최대 토큰 한계를 채우며 긴장감 넘치는 이야기를 계속 써 내려갑니다.
*   **메타 작가 융합:** 현재 서사에 가장 적합한 새로운 작가(또는 SF, 누아르, 시대극 등의 장르)의 문체를 자율적으로 선택하여, 기존 게이고 스타일과 30% 비율로 융합합니다.
*   **논리적 정밀성 프로토콜:** 추상적인 비유를 **미세 관찰**, **물리적 증거**, **엄격한 인과율**로 대체합니다. '감정'보다는 '사실'과 '추론'을 우선시합니다.
*   **고속 가독성:** 단문 위주의 문장, 건조한 서술, 빠른 템포의 대화를 사용하여 독자가 페이지를 멈출 수 없게 만드는 몰입감을 제공합니다.
*   **시그니처 서식:** 모든 대화에 인용문 들여쓰기(`>`)와 일본식 겹낫표(`「 」`)를 사용하는 독특한 시각적 스타일을 적용합니다.
*   **엄격한 독창성 원칙:** 작가의 방법론(동기 중심 서사, 반전 설계)만 계승할 뿐, 실제 작품의 탐정 캐릭터나 설정을 일절 사용하지 않고 100% 독창적인 이야기를 생성합니다.
*   **특수 보고서 모드 (액션 프로토콜):** 소설 집필과 별개로, 정확히 300자 분량의 60개 섹션으로 구성된 구조화된 보고서를 생성하는 고도로 제약된 보조 기능입니다.

## 🚀 사용 방법

### 1. 시스템 프롬프트 적용
제공된 `System Prompt` (Project K) 전체를 모델의 시스템 설정에 입력합니다.

### 2. 트리거 명령어
간단한 명령어로 서사의 흐름을 제어합니다.

| 명령어 | 기능 | 동작 방식 |
| :--- | :--- | :--- |
| **"[주제]"** | 서사 시작 | 주제를 기반으로 새로운 오리지널 소설의 1장을 시작합니다. |
| **"계속"** | 이어 쓰기 | 이전 응답이 끊긴 부분부터 정확히 이어서 집필을 재개합니다. AI가 새로운 서브 작가 스타일을 도입합니다. |
| **(스타일 제안)** | 융합 유도 | 다음 턴에 적용할 스타일을 제안할 수 있습니다. _예: "(사이버펑크 풍으로) 계속"_ |
| **(보고서 활성화)** | 보고서 모드 전환 | 액션 프로토콜을 사용하려면 분석/보고 관련 주제로 시작하세요. _예: "현재 경제 상황을 분석해 줘."_ |

## 🛠️ 기술적 컨셉

이 시스템은 다음과 같은 핵심 문학 및 구조적 프로토콜 위에 구축되어 있습니다.

1.  **서사 톤의 5가지 기둥:** 이야기의 논리를 관장하는 규칙 세트 (영화적 리얼리즘, 서스펜스의 서브텍스트, 엄격한 인과율, 와이던잇(Whydunit), 사회적 맥락).
2.  **미세 관찰 엔진:** 순간을 감각적 인상이 아닌 검증 가능한 사실(정확한 시간, 물체의 물리적 상태, 무의식적 반사 행동)로 해체합니다.
3.  **가변적 종결 아키텍처:** 매번 비장한 결말을 맺는 것을 방지하기 위해, 행동, 대화, 건조한 관찰 도중에 서술을 뚝 끊는 다양한 종료 방식을 무작위로 적용합니다.
4.  **액션 프로토콜:** 비(非)서사적 작업을 위한 별도의 엄격한 서식 규칙으로, 구조와 글자 수의 극단적인 정확성을 요구합니다.
